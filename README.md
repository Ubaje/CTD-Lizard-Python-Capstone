# ⚾ MLB Historical Dashboard — Capstone Project

A four-program data pipeline that scrapes MLB historical statistics from Baseball Almanac, cleans and stores the data in SQLite, supports interactive command-line querying, and presents an interactive dashboard built with Streamlit and Plotly.

> **Lesson 14 — Web Scraping and Dashboard Project**
> Course assignment demonstrating Selenium scraping, data cleaning, SQLite storage, JOIN-based querying, and interactive data visualization.

## 🔄 Running the Full Pipeline

Run the four programs **in order**. Each program depends on the output of the previous one.

---

### Step 1 — Scrape Data (`1_scraper.py`)

Scrapes AL and NL yearly review pages from Baseball Almanac using Selenium.
Saves two CSV files: a long-format stat leaders table and a pivoted player stats table.

```bash
python 1_scraper.py                        # default: 2000–2023
python 1_scraper.py --start 2010 --end 2023   # custom year range
```

**What it does:**
- Launches headless Chrome with a realistic User-Agent header
- Scrapes both AL and NL pages for each year
- Filters out junk rows (division headers, team win totals, section labels)
- Normalizes franchise names across historical rebrandings (e.g. `"Anaheim Angels"` → `"Angels"`)
- Logs before/after row counts at each cleaning stage

**Outputs:** `data/events.csv`, `data/players.csv`

---

### Step 2 — Import to Database (`2_db_import.py`)

Loads the CSVs into a SQLite database. Each CSV becomes a separate table.
Applies an independent layer of data cleaning before insert.

```bash
python 2_db_import.py --data-dir ./data --db ./db/mlb.db
```

**What it does:**
- Validates CSVs exist before running
- Runs table-specific cleaners (`clean_events_df`, `clean_players_df`) — stat whitelist, junk row removal, team name normalization
- Infers column types dynamically (no hardcoded schema)
- Builds indexes on `year`, `player`, `team`, `statistic`, `league`
- Logs row counts before and after cleaning, and verifies insert count matches

**Output:** `db/mlb.db`

---

### Step 3 — Query the Database (`3_query.py`)

Interactive command-line tool for querying the database.
Supports preset named queries (including JOINs), free-form SQL, and filter shortcuts.

```bash
python 3_query.py                              # interactive menu
python 3_query.py --list-queries               # show all named queries
python 3_query.py --query top_hr               # run a named query
python 3_query.py --year 2015                  # filter by year
python 3_query.py --player "Trout"             # filter by player name
python 3_query.py --statistic "Home Runs"      # filter events by stat
python 3_query.py --sql "SELECT * FROM events WHERE year = 2001;"
```

**Available named queries:**

| Query | Description |
|---|---|
| `top_hr` | Top 20 single-season home run totals |
| `top_avg` | Top 20 single-season batting averages |
| `hr_by_league` | Avg HR per season by league — JOIN: players × events |
| `stat_leaders_by_year` | All stat leaders per year — JOIN: events × players |
| `team_stat_leaders` | Teams with most stat leader appearances — JOIN: events × players |
| `player_dominance` | Players who led the most categories — JOIN: events × players |
| `decade_summary` | Average offensive stats grouped by decade |
| `events_by_year` | All stat leaders from the events table |
| `al_vs_nl_hr` | AL vs NL HR leader side-by-side per year — self-JOIN on players |
| `top_hitters_per_team` | Best batting average per team across all years |

**Interactive menu options:**
- `1–10` — run a preset query
- `s` — enter custom SQL
- `y` — filter by year
- `p` — filter by player name
- `a` — filter by stat category
- `m` — reprint the menu
- `q` — quit

### Step 4 — Launch the Dashboard (`4_dashboard.py`)

Interactive web dashboard built with Streamlit and Plotly.

```bash
streamlit run 4_dashboard.py
```

Opens at `http://localhost:8501`

**Visualizations:**

| Chart | Description |
|---|---|
| Peak HR per Season | Line chart showing the highest single-season HR total per year |
| Batting Average Distribution | Histogram of BA values with mean line |
| Player Performance Scatter | Configurable axes — any two stats, one point per player |
| Stat Leader Appearances by Team | Horizontal bar chart of which teams dominate stat categories |
| Stats by Decade | Grouped bar + line chart of avg HR and BA by decade |

**Sidebar filters:**

| Filter | Affects |
|---|---|
| Season Range (slider) | All charts, KPI strip, Stat Leaders table |
| League | All charts, Stat Leaders table |
| Teams | HR Trend, BA Distribution, Scatter, Decade Summary |
| Scatter X / Y Axis | Scatter chart only |
| Stat Category | Stat Leaders table only |

## 🗃️ Database Schema

Schema is inferred dynamically from the CSV columns — no hardcoded DDL.
The tables below reflect the typical output from the scraper.

**`events` table** — one row per stat leader per year per league
```
year        INTEGER   -- season year
league      TEXT      -- AL or NL
statistic   TEXT      -- stat category (e.g. "Home Runs", "ERA")
player      TEXT      -- player name
team        TEXT      -- canonical franchise name (e.g. "Yankees")
value       REAL      -- stat value
```

**`players` table** — one row per player per year (pivoted from events)
```
year          INTEGER
league        TEXT
player_name   TEXT
team          TEXT
batting_avg   REAL
home_runs     REAL
hits          REAL
stolen_bases  REAL
era           REAL
wins          REAL
so            REAL
```

## 🧹 Data Cleaning

Cleaning happens at **two independent stages** — in the scraper and again in the importer.

**Stage 1 — Scraper (`1_scraper.py`)**
- Junk rows filtered at parse time: division headers (`East`, `West`, `A.L.`), team win-total rows, rows with non-numeric or zero values
- Stat whitelist: only ~18 known real stat categories are kept
- Franchise normalization: all historical name variants mapped to one canonical name

**Stage 2 — Importer (`2_db_import.py`)**
- Same stat whitelist and junk filters applied independently as a safety net
- Same `TEAM_NORM` mapping applied so data is clean even if CSVs were edited externally
- Column types inferred: whole numbers → `INTEGER`, decimals → `REAL`, text → `TEXT`
- Before/after row counts logged for both tables

## ⚠️ Scraping Notes

- Targets [Baseball Almanac](https://www.baseball-almanac.com) yearly review pages
- Headless Chrome with realistic `User-Agent` header to avoid bot detection
- Polite random delays of 1.5–3.0s between requests
- Exponential backoff retry logic (up to 3 attempts) on connection errors
- Both AL and NL pages scraped per year from separate URLs
